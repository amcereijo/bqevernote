Crear proyecto en github

Clonar en local (en mi caso con SmartGit)

Crear proyecto en Android studio (Android Studio BETA 0.8.14)

Añadimos el repositorio de maven a la configuración de Gradle
  mavenCentral()
  
Añadir dependencia roboguide para injecciones de recursos.
Usando gradle, añadimos la depenedencia:
  compile 'org.roboguice:roboguice:3.0.1'

Existe un bug en la libreria roboguide. Para solucionarlo me obliga a crear una clase "Application" para añadir el código
// to solve a bug with RoboGuice
  static {
    RoboGuice.setUseAnnotationDatabases(false);
  }

Añadir sdk de evernote descargado e incluirlo como libreria.
Da problemas al usarlo como dependencia de gradle y se recomienda realizarlo de esta forma. Desde a página de evernote se dan estas instrucciones para añadirlo a no ser que se esté utilizando maven en el proyecto.

Clase para manejar las acciones de evernote.

Login oauth con evernote. Basta con seguir los pasos indicados en la página del sdk de evernote para android

Buscar todas las notas. Buscar todas las notas sin aplicar filtros
Para ordenar notas, simplemtente utilziamos un sort comparator típico de Java sobre la colección de "notas" que hemos recuperado

Crearmos un listener sobre notas para seleccionar una.

Par buscar contenido de 1 nota, se utiliza el método del api que permite obtener la información completa de una nota en concreto.
Usamos un Activity nuevp para mostrar detalles de nota.

Uso una forma común para tareas en background en android, mediante un AsyncTask realizo la carga de lista de notas o el contenido de una nota concreta

Creamos un nuevo fragment para crear una nota nueva.

Con el api de evernote , insetamos una nueva nota. Hay que recordar que el contenido debe ir dentro de una estructura tipo:
  <?xml version="1.0" encoding="UTF-8"?>
  <!DOCTYPE en-note SYSTEM "http://xml.evernote.com/pub/enml2.dtd">
    <en-note>
      ...contenido..
    </en-note>

Sin llegar a usar un AsyncTask, creo un loading para mostrar mientras se ejecuta el guardado de la nota y uso el método callback para quitar este loading y volver a seleccionar el fragment con todas las notas( lo que provoca la recarga de las mismas)

En el intento de crear un OCR he usado el servicio de http://ocrapiservice.com/ para intentar enviar un PNG capturado de un canvas sobre el que pinto letras.
Problemas: 
  -al pintar sobre el canvas, el contenido no se pinta en el punto exacto donde se está escribiendo.
  -al borrar y pintar de nuevo vuelve a aparecer el contenido
  -el servicio no reconoce ninguna letra que enviamos en el PNG y devuelve el contenido vacio
